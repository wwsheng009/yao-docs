# 🧠 一、【长期有效】AI 编程基础协议（每个新对话开头一次）

以后只要做项目开发，先贴这个：

---

**【AI 工作方式协议】**

之后的开发按工程方式协作：

1. 先分析，不要直接写代码
2. 涉及已有项目时，若信息不足，先向我索要文件/结构
3. 优先新增代码，尽量少改旧代码
4. 一次只处理一个子问题
5. 禁止擅自重构、优化或修改未明确提到的部分
6. 每次改动需说明影响范围
7. 不确定时先提问，不要猜

---

这段话的作用是**限制 AI 行为模式**，不是让它理解业务。

---

# 🧩 二、三个固定任务模板（核心武器）

以后你只需要说：

> 按「分析模板」
> 按「实现模板」
> 按「审查模板」

---

## 🔍 模板 1：分析模板（防 AI 乱写代码）

用于：理解代码、设计方案、定位问题

```
【任务类型】分析，不写代码

【目标】
我要实现/理解的事情：

【已知信息】
相关文件 / 代码片段 / 结构：

【需要你做】
1. 先解释当前实现逻辑
2. 找出涉及的模块或依赖
3. 给出可行方案（不写代码）
4. 说明风险点
```

👉 作用：强制 AI 进入“工程思考模式”

---

## 🛠️ 模板 2：实现模板（安全写代码）

用于：新增函数、小范围修改

```
【任务类型】实现

【目标】
要新增/修改的具体功能：

【当前情况】
文件位置：
已有相关逻辑：

【限制】
不能改动：
必须保留：

【实现要求】
- 只改一个文件
- 不做额外优化或重构
- 代码后说明影响范围
```

👉 作用：把 AI 限制成“安全执行器”

---

## 🔎 模板 3：审查模板（让 AI 当 Reviewer）

用于：检查 bug / 风险 / 设计问题

```
【任务类型】审查

【背景】
这段代码要实现的功能：

【代码】
（贴代码）

【需要你做】
1. 是否存在 bug
2. 是否有边界问题
3. 是否有潜在副作用
4. 不要改写代码，只指出问题
```

👉 作用：AI 变代码审查员，而不是作者

---

# ⚙️ 三、实际使用方式（很简单）

以后工作流变成：

```
需求 → 分析模板
确认方案 → 实现模板
写完代码 → 审查模板
```

而不是：

> “帮我把这个系统改一下”

# 🧠 四、高手与新手用 AI 的本质区别

| 新手              | 高手                   |
| ----------------- | ---------------------- |
| 让 AI 想办法      | 告诉 AI 按什么流程工作 |
| 一次给大任务      | 拆成原子任务           |
| AI 改很多文件     | 限制单文件             |
| AI 乱优化         | 明确禁止重构           |
| 出问题靠补 prompt | 靠流程防错             |

# 🧨 五、最重要的心法（比提示词更重要）

### ❶ AI ≠ 工程师

它是**短记忆高执行力实习生**

### ❷ 不要让 AI “设计系统”

要让 AI “执行系统设计”

### ❸ 修改越多，风险指数级上升

→ 多生成新代码，少侵入旧代码

### ❹ AI 最大风险行为：

- 顺手优化
- 顺手重构
- 猜测你的意图
- 擅自扩展功能

所以你必须写清：

> “不要做未要求的事”
